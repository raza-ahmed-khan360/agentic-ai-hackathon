## Corrected Approach

### Old (Incorrect) Approach ❌
- Manually create template files in `.specify/templates/`
- Users copy and fill in templates manually
- Error-prone and inconsistent

### New (Correct) Approach ✅
- Use **Spec-Kit Plus commands** to generate all documents
- Available commands (configured in `.gemini/commands/`):
  - `/sp.phr` - Generate Prompt History Records
  - `/sp.specify` - Generate specifications
  - `/sp.plan` - Generate implementation plans
  - `/sp.checklist` - Generate checklists
  - `/sp.adr` - Generate architectural decisions
  - And 6 more standard commands

## How to Generate Documents Going Forward

### Generate a PHR:
```bash
/sp.phr --title "My Work Summary" --stage general
```

### Generate a Specification:
```bash
/sp.specify --feature my-feature --title "Feature Specification"
```

### Generate a Plan:
```bash
/sp.plan --spec-file specs/001-feature/spec.md
```

### Generate a Checklist:
```bash
/sp.checklist --feature my-feature --type requirements
```
